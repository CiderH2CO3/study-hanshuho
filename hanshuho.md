# 1. 本人確認（KYC）の全体像

---

# 1. 本人確認（KYC）の全体像

### 顧客確認（CDD）の重要性
- 特定事業者は、取引の開始時に顧客の本人特定事項を確認する義務があります。
- マネー・ローンダリングおよびテロ資金供与対策（AML/CFT）の基盤となります。

### 本人確認の三要素
1. **本人特定事項**: 氏名、住居、生年月日（自然人の場合）
2. **取引目的**: 取引を行う目的の確認
3. **職業・事業内容**: 顧客の属性確認

# 2. 準拠バージョンと改正予定

---

# 2. 準拠バージョンと改正予定

### 準拠している条文
- **犯罪による収益の移転防止に関する法律施行規則（第六条）**
- **2025年（令和7年）6月施行分まで反映済み**
  - 第六条第一項第一号「ル」（スマホ版マイナ）を含む最新の内容です。

### 改正のロードマップ（2025年〜2027年）

| 時期 | 主な変更点 |
|------|------------|
| **2025年6月** | **方式「ル」の新設**: スマホ版マイナ（カード代替電磁的記録） |
| **2027年4月** | **大規模改正**: 写真撮影方式（ホ・リ・ハ・ニ）の廃止、IC読取の義務化。 |

---

# 2. 2027年4月 改正のポイント

2027年4月以降、**「券面の画像を送るだけ」の本人確認は原則廃止**され、ICチップ活用が必須となります。

### 主な変更点
- **券面撮影方式の廃止**: 偽造対策のため、スマホカメラによる撮影（ホ方式等）が認められなくなります。
- **ICチップ活用の義務化**: 対面・非対面を問わず、ICチップ情報の読み取りが原則必須となります。
- **方式記号の再編**: 大幅な条文番号の振り直しが行われます（例：JPKIがワ→ヌ等）。

# 方式「リ」と「ヌ」の違い

どちらも「書類の写し」を利用しますが、**「誰が」**送付し、**「誰を」**確認するかが異なります。

| 項目 | 方式「リ」 | 方式「ヌ」 |
|:---|:---|:---|
| **対象者** | 個人（一般顧客） | 法人の被用者等（従業員など） |
| **送付者** | 本人 | その者が所属する法人 |
| **書類の数** | 写し**2点**（または写し1点+補完1点） | 写し**1点** + 法人による証明 |
| **2027年改正** | **廃止**（成りすましリスク高） | **存続**（新「ヘ」方式） |

- **ポイント**: 方式「ヌ」は、法人が「この者は当社の社員です」と保証するプロセスが含まれるため、個人単独の「リ」よりも信頼度が高いとみなされています。

# 方式「リ」と「ヌ」の違い

- **ポイント**: ヌ方式は、法人が身元をある程度保証している（給与振込等）ため、リ方式よりも簡素化（書類1点）されています。

# 方式「ワ」「カ」「ヨ」の違い

これらはすべて「電子署名」を利用するデジタル完結方式ですが、**「どの法律に基づく証明書か」**が異なります。

| 項目 | 方式「カ」 | 方式「ワ」 | 方式「ヨ」 |
|:---|:---|:---|:---|
| **通称** | **公的個人認証 (JPKI)** | **認定電子署名** | **特定電子署名** |
| **根拠法** | 公的個人認証法 | 電子署名法（認定） | 犯収法施行規則 |
| **発行体** | J-LIS（地方公共団体） | 民間認定事業者（セコム等） | 認定を受けた特定事業者 |
| **主な証明書** | マイナンバーカード | 商業登記用電子証明書など | 特定のeKYC用証明書 |
| **2027年改正** | **新「ヌ」方式**へ | **新「リ」方式**へ | **新「ル」方式**へ |

- **実務上の選択**: 一般個人向けには、普及率の高い**「カ（JPKI）」**が最も一般的です。

# 【三号ホ】法人の本人確認（電子証明書）

.left-column[
### 概要
登記官が作成した**電子証明書**（商業登記電子証明書）の送信を受ける方法。

### 特徴
- 法人の電子署名をシステムで自動検証可能。
- 完全オンライン完結の法人KYCを実現。
]

.right-column[
### フロー
```mermaid
sequenceDiagram
    participant Corp as 法人(App)
    participant Biz as 事業者
    participant MOJ as 法務省
    Corp->>Biz: 電子署名 + 証明書
    Biz->>Biz: 有効性検証
    Biz-->>Corp: 完了
```
]

# 4.4 外国人の本人確認

# 【二号】外国人の本人確認

.left-column[
### 概要
法第四条第一項第一号に規定する**外国人**（特定の取引を行う者）の確認方法。

### 確認書類
- **旅券（パスポート）**
- **乗員手帳**
- **船舶観光上陸許可書**
]

.right-column[
### フロー
```mermaid
graph TD
    User[外国人顧客] -->|提示| Staff[窓口担当者]
    Staff --> Check1[旅券等の<br/>有効性]
    Staff --> Check2[氏名・<br/>生年月日]
    Staff --> Check3[上陸許可の<br/>確認]
    Check1 & Check2 & Check3 --> Done[確認完了]
```
]

# 5. 補完書類と実務上の留意点

.left-column[
### 【二項】補完書類
本人確認書類に**「現在の住居」**の記載がない場合に追加する書類。

### 主な種類
- 納税証明書、領収書
- 社会保険料の領収書
- 公共料金の領収書
]

.right-column[
### ルール
```mermaid
graph TD
    Sub[補完書類] --> TypeA[発行日があるもの]
    Sub --> TypeB[有効期限があるもの]
    TypeA -->|6ヶ月以内| OK[有効]
    TypeB -->|提示時に有効| OK
```
]

---

# 【三項・四項】営業所送付・役職員による交付

.left-column[
### 営業所への送付（三項）
- 法人顧客の場合、本店等ではなく**営業所**宛に取引関係文書を送付できる。

### 役職員による交付（四項）
- 郵送に代えて、事業者の**役職員が直接赴いて**手渡しする方法。
]

.right-column[
### フロー
```mermaid
sequenceDiagram
    participant Biz as 事業者
    participant Staff as 担当者
    participant User as 顧客
    Staff->>User: 訪問
    User->>Staff: 書類提示
    Staff->>User: 取引関係文書手渡し
```
]

# 【付録】2027年4月 改正前後の方式対照表(1/3)

| 現行方式 | 改正後 | 概要 | 変更内容・備考 |
|:---:|:---:|:---|:---|
| **イ** | **イ** | 写真付き提示（対面） | **ICチップ読取が義務化**。券面目視のみは不可。 |
| **ロ** | **ロ** | 写真なし提示＋郵送 | ロ(1)ICなし写真IDまたは住民票(原本)、ロ(2)非写真ICに分化。 |
| **ハ** | **廃止** | 書類2点提示（対面） | **廃止**。健康保険証の廃止等に伴い整理。 |
| **ニ** | **廃止** | 1点提示＋写し送付 | **廃止**。 |
| **ホ** | **廃止** | 容貌 ＋ 書類撮影 | **廃止**。券面の写真撮影（厚み等）による確認は不可に。 |
| **ヘ** | **ハ** | 容貌 ＋ ICチップ | **新「ハ」方式**。券面撮影が廃止され、IC読取が必須化。 |
| **ト** | **ニ** | IC読取 ＋ API/振込 | **新「ニ」方式**。券面撮影が廃止され、IC読取または原本が必須化。 |
